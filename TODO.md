# TogetherToTheSpire - 产品路线图

## 产品目标

把这个项目做成一个面向服务器管理员的“一键部署 + 交互式发放玩家配置”的工具。

核心场景：
- 在一台主流 Linux 服务器上快速部署 WireGuard
- 部署完成后，交互式询问有几个人要联机
- 自动批量创建 peer
- 每个玩家单独生成配置文件和二维码
- 明确告诉用户要开放哪些端口，以及服务器侧怎么检查
- 支持通过固定的远程入口一键安装，不要求用户先手动 git clone

产品定位：
- 不是通用 VPN 管理器
- 不是给普通玩家用的客户端
- 是“杀戮尖塔广域网联机专用的服务器开局工具”

---

## 当前状态

已具备：
- 服务端初始化的基础能力
- 单个 peer 的生成能力
- 客户端配置文件和二维码生成
- 等待握手的基础检测

缺少：
- 统一入口脚本
- 多发行版安装支持
- 多玩家批量交互
- 防火墙 / 安全组提示
- 安装完成后的说明文档
- 更完整的错误处理和回滚

---

## MVP

目标：让一个懂服务器的人，能在尽量少的输入下把环境搭起来并开始联机。

### 1. 统一安装入口

- [ ] 提供 `wg-setup.sh` 作为唯一入口
- [ ] 支持 `curl -fsSL <raw-url> | sudo bash`
- [ ] 支持下载后执行
- [ ] 启动时检测 root 权限
- [ ] 启动时检测常见依赖是否存在
- [ ] 入口增加交互菜单，支持 `setup / add-peer / remove-peer / status / remove-env`
- [ ] 安装后仍然保留一个本地可执行入口，方便后续管理

### 2. 主流发行版支持

- [ ] 检测 Ubuntu / Debian
- [ ] 检测 CentOS / Rocky / AlmaLinux
- [ ] Ubuntu / Debian 使用 `apt`
- [ ] RHEL 系使用 `dnf`
- [ ] 对不支持的系统给出明确退出信息

### 3. 基础 WireGuard 部署

- [ ] 自动检测默认网卡
- [ ] 安装 `wireguard`、`qrencode`、`curl`
- [ ] 开启 `net.ipv4.ip_forward`
- [ ] 生成服务端密钥对
- [ ] 只把服务端密钥存到 `/etc/wireguard/`
- [ ] 写入 `wg0.conf`
- [ ] 配置 NAT 转发规则
- [ ] 启动 `wg-quick@wg0`
- [ ] 设置开机自启

### 4. 部署后提示

- [ ] 输出公网 IP 和端口
- [ ] 输出 VPN 网段
- [ ] 明确提示需要放行 UDP 端口
- [ ] 说明云服务器安全组要放行的方向

### 5. 单个 peer 能用

- [ ] 保留 `wg-add-peer.sh` 作为独立脚本
- [ ] 保持 `sudo ./wg-add-peer.sh <名字> [超时秒数]` 可用
- [ ] 输出客户端配置文件
- [ ] 终端显示二维码
- [ ] 等待握手并给出结果

## V1

目标：把“能用”提升到“适合给朋友直接开局”。

### 1. 批量玩家交互

- [ ] 初始化完成后询问“几个人要联机”
- [ ] 校验人数范围，例如 2 - 5 
- [ ] 逐个询问玩家名字
- [ ] 提示用户并回车时使用默认名，`战士`、`猎宝`、`亡灵`、`储君`、`鸡煲`、`观者大人`
- [ ] 根据人数自动循环生成 peer

### 2. 发放体验

- [ ] 每个玩家生成完后立刻展示二维码
- [ ] 二维码上方显示玩家名和 VPN IP
- [ ] 二维码下方提示“请 XXX 用 WireGuard 扫描”
- [ ] 每个二维码之间暂停，按回车继续
- [ ] 同时保存每个玩家的 `.conf`

### 3. 握手确认

- [ ] 批量检测所有 peer 是否握手成功
- [ ] 输出已连接 / 未连接状态
- [ ] 未连接时提示优先排查安全组和防火墙
- [ ] `status` 命令支持实时查看服务状态、peer 列表和握手情况

### 4. 防火墙兼容

- [ ] 检测 `ufw`
- [ ] 检测 `firewalld`
- [ ] 如已启用则自动放行 UDP 端口
- [ ] 如未启用则只给出手动提示

### 5. 说明文档

- [ ] 生成 `~/wg-clients/联机说明.txt`
- [ ] 写入杀戮尖塔联机步骤
- [ ] 写入 WireGuard 客户端下载入口
- [ ] 写入玩家扫码导入说明

---

## V2

目标：把工具做得更稳、更像一个可分发产品。

### 1. 远程分发

- [ ] 支持固定 raw 地址部署
- [ ] 明确推荐 `curl -fsSL ... | sudo bash`
- [ ] 支持从 release 下载
- [ ] 如果支持用户自定义 git 地址，要加来源校验和安全提示

### 2. 可靠性

- [ ] 安装前做系统检查
- [ ] 配置写入前做备份
- [ ] peer 追加前做重名检查
- [ ] 失败时输出清晰错误
- [ ] 关键步骤增加回滚思路

### 3. 产品边界

- [ ] 明确不做 IPv6
- [ ] 明确不做多接口
- [ ] 明确不做卸载脚本，或单独列入后续版本
- [ ] 明确支持范围只覆盖杀戮尖塔联机场景

### 4. 运维可见性

- [ ] 输出所有玩家的客户端 IP 列表
- [ ] 输出 WireGuard 服务状态
- [ ] 输出当前已配置 peer 数量
- [ ] 记录初始化时间和部署结果

## 现有脚本整理

### `wg-init.sh`

建议逐步收敛为：
- 初始化逻辑
- 调用安装逻辑
- 调用配置逻辑
- 调用 `wg-add-peer.sh` 或批量 peer 流程

当前需要优先修正：
- 只支持 `apt`
- 服务端密钥暂时落在当前目录
- 还没有统一入口脚本

### `wg-add-peer.sh`

建议保留为独立脚本。

它已经覆盖：
- 生成 peer 密钥
- 分配 IP
- 写入客户端配置
- 生成二维码
- 等待握手

后续需要统一：
- 和初始化脚本里的行为一致
- 端口 / 网段读取方式一致
- 输出格式一致

## 用户入口建议

推荐的安装方式：

```bash
curl -fsSL https://raw.githubusercontent.com/jesse-ux/TogetherToTheSpire/main/wg-setup.sh | sudo bash
```

备选方式：

```bash
wget https://raw.githubusercontent.com/jesse-ux/TogetherToTheSpire/main/wg-setup.sh
sudo bash wg-setup.sh
```

不推荐：
- 先要求用户 git clone 再执行
- 让用户自己拼接内部脚本路径

## 版本优先级

1. 先做 MVP，把单机部署和单个 peer 跑顺
2. 再做 V1，把人数询问和批量发放补齐
3. 最后做 V2，把分发、校验和可靠性收敛
